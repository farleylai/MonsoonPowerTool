Essential Gradle

11 June 2016

Gradle with IDE Caveats

  • Andorid Studio up to 2.2p3 does not use Gradle Test Runner w/ passed jvmargs and system properties

  • IntelliJ 2016.1.3 works as expected

Performance Optimizations

There are several incubating performance improvedment settings in ~/.gradle.properties as follows.

org.gradle.parallel=true
org.gradle.daemon=true
org.gradle.configureondemand=true
org.gradle.jvmargs=-Xms256m -Xmx1024m

Moreover, run gradle with --offline or set the option in IDE also improve the time.

Build Lifecycle

Build Phases

  • Initialization to create the root project and subprojects

    • settings.gradle is processed to create project objects. A multiproject build must have a settings.gradle file in the root project of the multiproject hierarchy but is not required for a single-project build.

      • include: 'p1:p2:p3' added projects p1, p2 and p3

      • Default to search master or ../ if not available

  • Configuration to configure the projects and determine the subset of tasks to execute

    • code in build.gradle and tasks but not in ordered action closures

  • Execution for executing the target tasks

    • action closures in order

Lifecycle Events

  • afterEvaluation { project[, state ] → }

  • tasks.whenTaskAdded { task → }

  • gradle.taskGraph.whenReady { taskGraph → }

  • gradle.taskGraph.beforeTask { task → }

  • gradle.taskGraph.afterTask { task[, state] → }

Organizing Build Logic

  • Multiple tasks share a common method

  • Multiple subprojects share the parent project

  • subprojects {}

  • project(:child) {}

  • apply from: other.gradle

Plug-in

  • Named domain objects for DSL extension

  • Plug-in Extension for DSL

  • Custom task based on DSL with default

  • Plug-in implementation to add tasks and extensions to project
