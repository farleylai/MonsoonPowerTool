Convolutional Neural Networks (CNN)

28 February 2016

Data Augmentation

  • Change the pixels as new training data

  • Horozontal flips

  • Random crops/scales for training and an averaged fixed set of cro ps for testing

  • Color jitter by applying PCA to RGB pixels

  • Random affine transformations

  • Lens distortions

  • Add stochastic noises to prevent overfitting with model ensembles

Transfer Learning

  • Train on pre-trained models such as ImageNet for reusing learned features

  • Small datasets

    • Freeze lower level convolution layers while only training the FC layers

    • Plus some more new convolution layers to train on larger datasets

    • 1/10th of the original learning rates in fine-tuning top layers

    • 1/100th on intermediate layers

  • Find a vert large dataset of similar data and train a big ConvNet for use on the target dataset

Convolution Practices

  • A smaller filter size with more nonlinear layers in between

    • fewer parameters

    • fewer floating point operations

  • Practices

    • 1x1 bottleneck conv reduces the dimension

    • 3x3 filters on reduced dimension

    • 1x1 filter to restore the dimension

  • GoogLeNet Inception module

    • 1x3 or 3x1 filters for asymmetric ConvNet building blocks

  • im2col to turn convolutions into matrix multiplications

    • Expand each receptive field as matrix columns

    • Add mini-batch examples if any

  • Apply FFT and iFFT to compute dot in the frequency space on larger than 3x3 filters

Implementation Performance

  • Spread mini-batch across GPUs

  • Data parallelism on lower convolutional layers

  • Model parallelism on FC layers among GPUs

  • 16-bit half precision with stochastic rounding

  • 10-bit activations and 12-bit parameter updates

  • 1-bit activations and weights with more bit precision for gradients
