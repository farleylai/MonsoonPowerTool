Reinforcement Learning

16 January 2016

Terminology

task

an instance of the reinforcement learning problem

Markov property

the next state only depends on the current state

Markov Decision Process (MDP)

a reinforcement learning task satisfying the Markov property

finite MDP

state and action spaces are finite

Elements of Reinforcement Learning

  • Policy:: mapping to probability of action \(a\) taken in state \(s\)

    • function

    • table

    • search

    • stochastic

  • Reward Function

    • goal of the problem to maximize in the long run

    • state-action pair to a reward value

    • immediate and defining features

  • Value Function

    • the total amount of reward expected to accumulate over the future from the current state

    • predicted and secondary

  • Environment Model

    • given a state and action, predict the next state and next reward

    • used for planning on deciding a course of future action by considering possible future situations

    • from try-and-error to dynamic programming

Evolutionary Methods

  • Search in a small state space

  • Hard to accurately sense the state of the environment

  • No interaction with the environment

Impractical Assumptions

  • accurate dynamics of the environment

  • sufficient computational resources including memory to compute the solution

  • Markov property

Dynamic Programming

Monte Carlo (MC)

Time Difference (TD)

  • Monte Carlo + dynamic programming

  • Generalized policy iteration (GPI) for the control problem to find an optmial policy

  • Policy evaluation or prediction probability of the value function

    • use experience following policy \(pi\) to update the value estimates for nonterinal states

    • Monte Carlo methods wait until the return following the visit is known

    • Constant-\(\alpha\) MC: return estimate += \(\alpha\)(actual return - return estimate)

    • TD only waits for the next time step to update both reward and value

    • TD estimate = MC sampling + DP bootstrapping
